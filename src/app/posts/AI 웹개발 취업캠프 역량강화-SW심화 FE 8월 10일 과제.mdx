# AI 웹개발 취업캠프 역량강화-SW심화 FE 8월 10일 과제
<time>2023-08-14</time>

## 요약
- 할 일 목록 애플리케이션을 개발했다. CRUD 고려하며 개발했고, 로컬스토리지를 활용했다. styled-components 라이브러리를 활용하여 나만의 스타일을 가진 할 일 목록을 만들었다.

## 실습 과정
### 상태 관리 세부 절차(CRUD)
- `useState` 훅을 이용하여 `todo`와 `todoId` 상태를 관리
    - `todo`는 할 일 목록을, `todoId`는 새로운 할 일의 고유 ID를 나타냄
#### 추가(Create)
- `handleSubmit` 함수는 할 일 목록에 새로운 할 일을 추가
    - 이때 새로운 할 일은 기존 할 일 목록인 `todo` 배열에 추가
        - `...todo` 문법은 기존 배열의 내용을 그대로 유지하면서 새로운 할 일 객체를 추가
    - `setTodoId`로 `todoId`를 증가시켜 새로운 할 일의 고유 ID를 생성
#### 읽기(Read)
- `map` 함수를 통해 `todo`에 저장된 값들을 반환하여 표시
#### 갱신(Update)
- `handleToggle` 함수는 특정 `todoId`를 가진 할 일의 완료 상태를 변경
    - `setTodo`를 사용하여 해당 `todoId`를 가진 할 일의 `todoDone` 속성 값을 반전시켜 갱신
#### 삭제(Delete)
- `handleDelete` 함수는 특정 `todoId`를 가진 할 일을 목록에서 삭제
    - `setTodo`를 사용하여 해당 `todoId`를 가진 할 일을 제외한 나머지 할 일로 배열 갱신
### 로컬스토리지 활용 방법
- 페이지가 로드될 때, `useEffect`를 이용하여 이전에 저장한 할 일 목록을 `localStorage`에서 획득
- 가져온 할 일 목록이 있다면, `setTodo`를 이용하여 `todo` 상태 초기화 및 마지막 할 일의 `todoId`를 계산하여 `setTodoId`로 설정
    - 이를 통해 새로운 할 일이 추가될 때 중복되지 않는 고유 ID를 생성
- 할 일 목록이 변경될 때마다, `useEffect`를 이용하여 변경된 할 일 목록을 `localStorage`에 저장
### 스타일 정의 방법
- styled-components 라이브러리를 사용
- CSS-in-JS 기능을 지원
- 태그드 템플릿 리터럴 문법을 지원하여 CSS 문법 그대로 작성 가능
- CSS 키프레임을 `keyframes` 함수로 정의하여 애니메이션 제작
