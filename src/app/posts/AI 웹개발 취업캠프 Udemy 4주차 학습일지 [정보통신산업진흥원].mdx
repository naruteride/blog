### JS Engine의 동작 방식
- <Image src={img1}
    alt="JS Engine의 동작 방식을 설명하기 위한 그림. Call Stack에 Main Context, asyncAdd() 함수가 있다. WebAPIs에 setTimeout()에 쌓인 cb() 콜백함수가 있다. Callback Queue는 비어있다. Event Loop는 Callback Queue에서 Call Stack으로 향하는 화살표 모양의 통로이다." />
- 그림의 상황에서 `cb()` 콜백함수가 `setTimeout()`의 시간이 끝날 때까지 WebAPIs에서 머문다.
- 그동안 Call Stack은 **비동기로 동작 중**이다.
- `setTimeout()`의 시간이 끝났다면, `cb()` 콜백함수는 Callback Queue로 넘어간다.
- Event Loop가 Call Stack를 계속 감시한다.
- Call Stack에 `Main Context`만 남았다면, `cb()` 콜백함수가 Event Loop에 의해 Call Stack으로 넘어가서 실행된다.
### Web APIs를 사용하는 함수
- **`document.querySelector()`**: 주어진 CSS 선택자를 사용하여 문서에서 요소 검색
- **`addEventListener()`**: 이벤트 리스너를 등록하여 특정 이벤트가 발생했을 때 실행할 함수 지정
- **`fetch()`**: 네트워크 요청을 보내고 응답을 받아오는데 사용되는 함수로, AJAX 요청을 처리
- **`setTimeout()`**: 일정 시간이 지난 후에 함수를 실행하기 위해 사용되는 타이머 함수
- **`setInterval()`**: 일정한 시간 간격으로 함수를 반복적으로 실행하기 위해 사용되는 타이머 함수
- **`localStorage`** 및 **`sessionStorage`**: 로컬 스토리지와 세션 스토리지에 데이터를 저장하거나 가져올 수 있는 API
- 외에 더 많은 함수가 존재한다.
